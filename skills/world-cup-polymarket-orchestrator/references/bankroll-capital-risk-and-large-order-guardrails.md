# Bankroll capital risk and large-order guardrails

## 触发场景

当主人要求世界杯/足球 Polymarket 下单、加仓、把某场提高到某个总额，或问“按总资本 X 算是否合理”时使用。

本规则补充 `proportional-staking-template.md`：`1份=10 USDC` 仍是普通研究仓位单位，但当主人给出总资本（如 10,000 USDC）或要求大额实盘时，必须同时按本金百分比做风险闸门。

## 资本百分比口径

默认计算：

```text
总敞口比例 = 当前未平仓总成本 / 总资本
单场比例 = 该比赛所有市场成本合计 / 总资本
单市场比例 = 单个 token/condition 成本 / 总资本
```

报告时必须同时给：

```text
金额 USDC
占总资本百分比
占当前未平仓总敞口百分比
```

## 仓位分级

以总资本 10,000 USDC 为例：

- 单场 1%-2.5%：合理/保守（100-250 USDC）
- 单场 2.5%-5%：积极（250-500 USDC）
- 单场 5%-10%：高风险重仓（500-1000 USDC）
- 单场 >10%：极高风险，默认不建议执行
- 总未平仓 10%-15%：偏积极
- 总未平仓 15%-25%：偏激进，停止新增或只做减仓/对冲
- 总未平仓 >25%：高风险，默认只允许减仓/止损/锁盈，除非主人明确二次授权

## 大额订单执行前必须先输出风险卡

当任一条件满足时，必须先输出风险卡并等主人明确确认：

- 单场计划金额 >= 总资本 5%
- 单场计划金额 >= 500 USDC
- 单笔订单 >= 300 USDC
- 当前总未平仓后将超过总资本 15%
- 用户要求“坚持下/重仓/满仓/梭哈/1000”这类高风险措辞

风险卡模板：

```text
当前总资本：X USDC
当前未平仓：Y USDC（Y%）
本次计划：Z USDC（Z%）
执行后未平仓：Y+Z USDC（...%）
单场集中度：...
建议：不建议 / 可小额 / 可执行但高风险
如果仍要执行，请回复：确认按 [具体金额+方向] 执行
```

注意：如果用户已经在同一轮明确回复“坚持下 1000 / 确认执行”，可执行，但最终报告必须标记“高风险 override”。

## 大额计划的默认降级方案

当用户给出大额（例如 1000 USDC）但未指定方向，先给 3 档：

```text
纪律版：总资本 2%-3%
积极版：总资本 3%-5%
激进版：总资本 5%-10%，需要确认
```

示例：总资本 10,000 USDC，用户说“给你 1000 去下注美国”：

```text
纪律版 200-300：主方向 60%，副方向 40%
积极版 500：主方向 60%，副方向 40%
激进版 1000：主方向 60%，副方向 40%，必须明确确认
```

## 组合相关性规则

同一场内的 moneyline、spread、totals 常高度相关。报告时必须说明相关性：

- “强队胜 + Under 2.5” = 小胜剧本，若弱队先进球，双腿会同时承压。
- “强队胜 + -1.5 + Over” = 大胜剧本，波动更大。
- “胜负 + 同队让深盘” = 方向高度重叠，不应都重仓。

大额组合建议：

- 主方向 50%-70%
- 副方向 30%-50%
- 让深盘/正确比分/半全场合计不超过该场 20%

## 执行后复核

大额订单完成后，必须做：

1. HK 出口确认。
2. 每个订单 status=matched/open/failed。
3. 用 `positions` 回读最终持仓；若 data-api 延迟导致 `avgPrice/initialValue=0`，等待 10-20 秒后复查。
4. 用 `activity` 交叉核对成交记录，注意 `activity.usdcSize` 可能包含费用/显示口径，与 CLOB `makingAmount` 不完全一致；报告以 CLOB 回执和 positions 成本为主，并说明口径。
5. 给出场级汇总：成本、当前估值、浮盈亏、占总资本比例。

## 禁止/提醒

- 不用“稳、必中、包赢”描述大额仓位。
- 用户从 1份=10 USDC 跳到 500/1000 USDC 时，必须指出这是从“份数下注”切换到“资本级仓位”。
- 当总未平仓已经接近或超过 20% 总资本，后续默认建议停止新增，只做赛前阵容复核、减仓、锁盈或止损。
