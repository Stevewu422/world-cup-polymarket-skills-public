# 世界杯每日动态预测

## 何时使用

当用户要求：
- 世界杯预测、明天比赛预测、今日复盘、下注方向、盘口策略；
- 每天根据比赛情况更新第二天预测；
- 足球大赛小组赛/淘汰赛胜平负、让球、大小球、仓位、风控建议。

本 skill 必须与 `sports-betting-risk-management` 的风险纪律兼容：不说稳赚、稳胆、必胜；所有结论都用概率、条件和仓位表达。

## 核心原则

1. **先核验赛程与时间**：不得凭记忆报赛程；必须查最新赛程、开球时间、比赛阶段，并换算北京时间。
2. **先复盘再预测**：根据最近 24 小时已完赛结果、球队真实表现、伤停、红黄牌、晋级形势和市场反应，修正原有判断。
3. **赔率决定下注，不等于预测**：若没有实时赔率，给“可买阈值/低于则放弃”；不强行下注。
4. **仓位纪律优先**：总资金按 100 单位；普通方向 0.5-1 单位，强价值 1-1.5 单位，单场上限 2 单位；当天亏损 5 单位停止；连续错 3 场停止。
5. **明确不碰什么**：高水冷门、赔率被压穿、阵容不确定、动机复杂、正确比分重仓、大串关，都要列为规避项。
6. **预测与下注分层**：先给比赛概率判断，再比较市场价格是否有价值；“看好会赢”和“值得下注”必须分开写。
7. **每场必须有反证**：A/B 级方向必须写最强反方观点、推翻条件和临场降级触发器；回答不了反证时，信心至少下调一级。
8. **用复盘校准模型**：赛后不只看输赢，还记录买入价、收盘价、CLV、模型概率偏差和错误原因，下一轮按错误类型修正。

## 每日工作流

### 0. 每日两次更新模式（cron 优化）

本 skill 支持每天北京时间 **04:00** 与 **16:00** 两次自动运行。两次输出必须分工明确，避免重复：

- **04:00 早盘版**：复盘过去 24 小时结果，生成未来 24-36 小时初版预测，标出需要临场复核的伤停、首发、盘口和 Polymarket 价格。
- **16:00 临场版**：复核当天剩余比赛，比较 04:00 判断是否需要上调/维持/降级/取消，输出最终可执行单、条件单和观望单。

自动任务必须先判断当前运行时段：

```text
如果当前北京时间在 03:00-07:00 → 输出“早盘预测更新”。
如果当前北京时间在 15:00-18:00 → 输出“临场执行更新”。
其他时间手动触发 → 按“即时更新”处理，并说明当前距离开赛还有多久。
```

详细 cron runbook 见：`references/twice-daily-cron-world-cup-runbook.md`。

预测评分矩阵、反证流程、CLV/校准复盘见：`references/match-prediction-scoring-matrix.md`。生成 A/B/C 评级前必须先按该矩阵做简要打分。

### 1. 信息核验

每天生成预测前，必须查并交叉核验：
- 明天/未来 24-36 小时比赛清单；
- 已完赛结果和关键事件：进球、红牌、伤病、轮换、控球/射门/xG（可得则用）；
- 小组积分/晋级形势/淘汰赛规则；
- 伤停、停赛、主力轮换、新闻发布会；
- 主流赔率/盘口方向：1X2、亚洲让球、大小球、DNB/平手盘；
- 天气、旅行、休息天数、场地因素（有明显影响时写）。
- Polymarket 当前概率、流动性、盘口是否过热，以及与主流赔率的差异；
- 至少一个反方信息源或反方逻辑：如果找不到可靠反方，也要写“未找到强反方，但仍需临场复核...”。

### 2. 昨日复盘模板

```text
昨日复盘：
- 命中/偏差：...
- 市场误判：...
- 买入价/收盘价/CLV：...
- 概率校准：模型概率是否高估/低估...
- 需要修正的模型假设：...
- 今日风险等级调整：保守 / 正常 / 降仓 / 停手观察
```

复盘重点不是自夸，而是找偏差：
- 强队名气是否被高估；
- 弱队防守韧性是否被低估；
- 小组赛第三轮动机是否变化；
- 淘汰赛 90 分钟市场与晋级市场是否混淆；
- 大小球是否受早球/红牌扰动。

### 3. 比赛分级

每场先分级，再给下注建议：

- **A级：可执行**：信息清晰、赔率有价值、阵容/动机支持；仓位 1-1.5 单位。
- **B级：小注/条件执行**：方向有倾向但赔率或阵容未确认；仓位 0.5-1 单位。
- **C级：只看不买**：盘口过热、赔率无价值、动机复杂、信息不完整。
- **娱乐单**：比分/半全场/串关只能 0.1-0.2 单位，不进入主仓位。

### 4. 每场输出字段

每场必须输出，且必须**专门针对每一场比赛**给出方便第二天下注的结论，不只写总览。

```text
比赛：A vs B
时间：北京时间 YYYY-MM-DD HH:MM
阶段：小组赛/淘汰赛/其他
风险等级：A/B/C

评分矩阵：
- 基本面：0-5
- 阵容/伤停：0-5
- 动机/赛程：0-5
- 市场价格：0-5
- 信息确定性：0-5
模型倾向概率：...%
市场隐含概率：...%
价值差：... 个百分点

结论：主队不败 / 客队受让 / 小球 / 观望 等
主方向：...
副方向：...
Polymarket 候选市场：市场标题 / Yes 或 No / 当前价格或概率（若可得）
Polymarket 点击链接：https://polymarket.com/event/... 或 https://polymarket.com/market/...
赔率阈值：例如 ≥1.80 才考虑；低于 1.65 放弃
仓位：... 单位
不碰：...
反证：最强反方观点 / 推翻条件 / 降级触发器
临场复核：首发、盘口、伤停、动机、天气
如果临场变化：...
```

### 4.1 Polymarket 链接要求

每日预测必须尽量给出每场可点击的 Polymarket 网页链接，方便主人第二天手动打开下注：

1. 对每场比赛，用球队英文名、赛事名、日期组合搜索 Polymarket/Gamma markets。
2. 优先给与该场比赛直接对应的 market/event 页面链接；不要只给 Polymarket 首页。
3. 优先用 Polymarket Gamma API 核验市场与价格：`https://gamma-api.polymarket.com/events?slug=<slug>`；页面链接通常可写为 `https://polymarket.com/sports/world-cup/<slug>`。
4. 常见直接市场与更多盘口可分别有两个 slug，例如 `<match-slug>` 与 `<match-slug>-more-markets`；必须尽量同时查主胜/平局/客胜和 spread、O/U、BTTS 等更多盘口。
5. 每个链接旁边必须写明：
   - 市场标题；
   - 建议方向：Yes / No / 观望；
   - 当前概率/价格（如果可查）；
   - 执行条件：例如价格低于/高于多少才考虑。
6. `outcomes` / `outcomePrices` 可能是 JSON 字符串；读取后要配对解析，不要只看网页标题。
7. 如果没找到直接市场，写：`Polymarket：未找到该场直接市场`，可补充搜索链接或相关赛事页，但不得伪造链接。
8. 链接只用于主人手动点击下注；Agent 不自动下单、不替主人确认交易。
9. 若检测到当前服务器/IP 受 Polymarket 地区限制，只说明“本机可能无法直接交易，但链接可供主人自行打开确认”，不要尝试规避限制。

详细 API 探测方法见 `references/polymarket-gamma-world-cup-market-probe.md`。

## 小组赛/淘汰赛修正规则

### 小组赛第一轮
- 降低强队穿盘预期；
- 优先小仓位、DNB、让球 0/+0.25/+0.5；
- 大小球需结合两队节奏，不盲目大球。

### 小组赛第二轮
- 重点看第一轮真实状态与市场过度反应；
- 被低估但表现稳定的队可考虑受让；
- 输球方若必须抢分，小心后防风险和大小球变化。

### 小组赛第三轮
- 动机优先于纸面实力；
- 已出线球队可能轮换；
- 同时开球、净胜球、默契平局风险必须写明；
- 不清晰则宁可观望。

### 淘汰赛
- 区分 90 分钟胜平负、晋级、让球、加时点球；
- 强弱差距不大时，90 分钟平局概率上升；
- 下注更偏 DNB、受让、小球、晋级方向，但仍需赔率阈值。

## 禁止事项

- 禁止承诺“稳赚、必中、稳胆、梭哈”。
- 禁止推荐翻倍追损、马丁格尔、越输越加。
- 禁止在无赛程核验、无赔率条件时给重仓建议。
- 禁止把娱乐比分单包装成主单。
- 禁止忽略时区，尤其是跨日比赛。

## 每日自动任务提示词要点

自动运行时，应：
1. 使用 Web 搜索/可靠来源核验赛程、结果、积分、伤停和赔率；
2. 先判断当前北京时间属于 04:00 早盘版、16:00 临场版，还是手动即时更新；
3. 如果赔率源不可访问，明确“暂无可靠实时赔率”，改用条件阈值；
4. 输出中文简洁版，结论先行，逐场给出主方向/副方向/仓位/不碰项/赔率阈值/Polymarket 链接；
5. 16:00 版本必须说明相对 04:00 版本：上调/维持/降级/取消；若没有早盘上下文，则说明“未读取到早盘版本，按临场独立判断”；
6. 如可用 TTS，则附 60 秒内语音摘要；
7. 若未来 24-36 小时没有世界杯/目标赛事比赛，输出“无比赛”，并给后续观察点，不强行预测。

推荐 cron prompt 必须写明：只做研究与手动下注参考，不自动下注；不使用稳赚/必中/稳胆；单日总亏损 5 单位停手，连续错 3 场停手。

## 交付验收

一次合格的预测必须满足：
- 每场都有时间、方向、仓位、赔率阈值、不碰项；
- 有昨日复盘或说明昨日无比赛；
- 有总投入上限和停手机制；
- 明确哪些是主单、条件单、观望；
- 不出现确定性盈利暗示。
