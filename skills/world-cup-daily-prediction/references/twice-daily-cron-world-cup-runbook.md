# 世界杯预测每日 2 次 cron Runbook

## 目的

每天北京时间 04:00 与 16:00 自动更新足球/世界杯预测。04:00 偏“早盘/隔夜复盘与未来 24-36 小时初版”，16:00 偏“临场前二次确认与执行清单”。

## 两个时段的分工

### 04:00 早盘版

重点：
1. 复盘过去 24 小时已完赛结果。
2. 更新小组/淘汰赛形势、伤停、红黄牌、球队动机。
3. 拉取未来 24-36 小时赛程。
4. 给出初版方向和赔率阈值。
5. 标出需要 16:00 再确认的临场风险。

输出标题：`世界杯早盘预测更新（北京时间 YYYY-MM-DD 04:00）`

### 16:00 临场版

重点：
1. 复核当天剩余比赛是否变更。
2. 重新查伤停、预计首发、官方首发（若已出）、盘口/赔率/Polymarket 价格变化。
3. 对 04:00 版本做“上调/维持/降级/取消”。
4. 输出最终可执行单、条件单、观望单。
5. 明确当日总仓位、单场上限、停手机制。

输出标题：`世界杯临场执行更新（北京时间 YYYY-MM-DD 16:00）`

## 必查数据源优先级

1. 官方赛程/赛事页：FIFA、UEFA、CONMEBOL、赛事官网。
2. 结果和技术统计：Sofascore、FotMob、ESPN、BBC Sport、Flashscore、FBref/xG（可得则用）。
3. 伤停和首发：球队官网、赛事官网、主流体育媒体、Sofascore/FotMob lineup。
4. 赔率：OddsPortal、Bet365/主流赔率聚合、Polymarket Gamma API。
5. 天气/场地：仅当极端天气、草皮、旅行距离明显影响时写。

如果某类源不可访问，必须写“暂无可靠实时数据”，并改用条件阈值，不得编造。

## Cron 输出要求

Telegram 友好、结论先行，不用表格。

固定结构：

```text
世界杯早盘/临场预测更新（北京时间 YYYY-MM-DD HH:MM）

结论：今天保守/正常/进攻；最多投入 X 单位；亏损到 Y 单位停手。

一、过去 24 小时复盘
- 命中/偏差：...
- 模型修正：...

二、未来 24-36 小时比赛总览
- A vs B：主方向 / 风险等级 / 仓位 / 是否需临场复核

三、逐场执行卡
1. A vs B
时间：北京时间 ...
风险等级：A/B/C
主方向：...
副方向：...
Polymarket：链接 / Yes-No / 价格 / 阈值；找不到则说明未找到
赔率阈值：...
仓位：...
不碰：...
临场变化处理：...

四、最终单
主单：...
条件单：...
观望/放弃：...

五、风控
不追单、不倍投、不重仓比分、不因为上一场输赢改仓位。
```

## 去重与更新规则

- 16:00 不是重复 04:00 内容，而是对早盘判断做变更说明。
- 如果无变化，写“较早盘无重大变化”，仍需给最终执行卡。
- 若无比赛，不强行预测：写明下一场比赛时间、重点观察变量、是否需要下一次 cron 再更新。

## 安全边界

- 只给研究和手动下注参考，不自动下单。
- 不提供必胜/稳赚语言。
- 单日总投入建议默认不超过 3-5 单位；信息不足时降到 1-2 单位或观望。
- 正确比分、半全场、大串关只能列娱乐单，0.1-0.2 单位，不进入主单。
